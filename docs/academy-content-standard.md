# Academy Content Standard

This repository is the canonical source for the Private Academy - Industrial GTM. Notion is the review and delivery layer, but this repo must hold the complete academy intelligence.

## Source Of Truth Rule

Every substantial content improvement starts in GitHub. Notion may improve navigation, page mentions, comments, and reading flow, but the instructional substance must live here first.

## Lesson Standard

Every lesson must teach an operator how to decide, build, critique, and run an industrial GTM artifact. A lesson is not acceptable if it only names a concept.

Each lesson must include:

1. **Operator promise:** what the learner will be able to decide or build.
2. **Source anchors:** master intelligence files or templates used.
3. **Concept explanation:** what the concept means in industrial GTM terms.
4. **Why it matters:** how it affects sales, RevOps, buyer enablement, pipeline, and leadership decisions.
5. **Industrial operating model:** the working system, rules, or sequence.
6. **Composite industrial example:** anonymized, category-based, and practical.
7. **Diagnostic questions:** questions that reveal whether the current system is weak.
8. **Individual exercise:** the actual artifact work.
9. **Group critique:** review prompts that force operator judgment.
10. **AI-assisted attempt:** a governed prompt or output requirement.
11. **Human correction:** what humans must reject or repair in AI output.
12. **Final artifact:** what the learner must leave with.
13. **RevOps translation:** fields, states, owners, dashboards, or cadence implications.
14. **Failure modes:** anti-patterns and red flags.
15. **Completion standard:** what field-ready work looks like.

## Page Standard

Every page in the academy should make clear whether it is:

- teaching a concept;
- giving a field guide;
- guiding a facilitator;
- guiding an AI agent;
- providing a template;
- providing a worked example;
- defining quality control;
- supporting release and review.

If a page references another real academy page, the Notion version must use a page mention or direct page hyperlink. In GitHub, use relative links.

## Industrial Intelligence Standard

Content must be grounded in industrial GTM realities:

- long-cycle buying;
- technical and conservative buyers;
- committee decision-making;
- specification, proof, validation, implementation, and procurement risk;
- low-volume but high-value pipeline;
- distributor, consultant, association, event, and sales-led distribution;
- account movement instead of vanity metrics;
- RevOps visibility into signals, owners, SLAs, and progression states;
- AI governance and human correction.

## Rejection Standard

Reject academy content if it:

- sounds like generic SaaS demand generation;
- teaches lead collection instead of demand creation, distribution, capture, and enablement;
- uses vanity metrics as primary proof;
- skips buying committee reality;
- ignores distribution;
- treats engagement as buying intent;
- lacks RevOps translation;
- gives AI the final word;
- uses named-company case studies or restricted inspiration.

## Notion Sync Standard

When GitHub content changes:

1. update the repo first;
2. run validation;
3. commit and push;
4. update the corresponding Notion page;
5. preserve Notion page map links and page mentions;
6. include the source file path and source commit where useful.

GitHub is the content system. Notion is the human review and delivery surface.
