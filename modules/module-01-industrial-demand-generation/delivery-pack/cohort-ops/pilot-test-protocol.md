# Pilot Test Protocol

Use this protocol before declaring Module 1 release-grade.

## Pilot Purpose

The pilot tests whether Module 1 changes operator behavior, not whether participants like the material.

## Cohort Setup

| Item | Standard |
|---|---|
| cohort size | 4-8 participants |
| roles | sales, marketing, RevOps, SME or technical, sponsor observer when possible |
| category | one anonymized manufacturing or export category |
| duration | one day intensive, two day workshop, or four half-days |
| AI use | allowed only with prompt library and correction log |
| final output | 90-day Industrial Demand Generation Pilot |

## Pre-Work

1. Choose a category.
2. Collect non-confidential notes on buying committee, current content, channels, and sales process.
3. Review [../START-HERE.md](../START-HERE.md).
4. Review [../flowcharts/00-module-visual-map.md](../flowcharts/00-module-visual-map.md).
5. Confirm no restricted real company names are used in examples or submissions.

## Live Test Agenda

| Phase | Test Question | Evidence |
|---|---|---|
| orientation | can participants explain demand generation as an operating system? | demand system diagnostic |
| buyer reality | can they map real roles and proof needs? | buying committee map |
| focus | can they narrow ICP and disqualify? | ICP scorecard |
| MOIN | can they convert uncertainty into buyer questions? | MOIN grid |
| content | can they connect assets to sales use and signals? | content plan |
| distribution | can they choose trust paths beyond publishing? | distribution map |
| routing | can they route signals without overreacting? | signal-routing table |
| cadence | can they govern the pilot weekly? | cadence board |

## AI-Assisted Test

Each cohort should run one AI-assisted attempt and one human correction cycle.

Required evidence:

- prompt used;
- AI assumptions;
- AI unknowns;
- human corrections;
- rejected or repaired sections;
- final artifact change.

## Pass Criteria

The pilot passes when:

- at least one capstone scores 80 or above;
- no capstone passes without signal routing, weekly cadence, and disqualification;
- participants can name what sales should do differently;
- RevOps can name the fields and dashboard needed;
- the facilitator captures at least five improvement notes.

## Failure Criteria

The pilot fails if:

- participants produce generic marketing plans;
- content planning stays format-first;
- signal routing becomes lead scoring;
- weekly cadence becomes campaign reporting;
- AI output is accepted without correction;
- confidentiality boundaries are violated.

## After The Pilot

1. Update [cohort-feedback-log.md](cohort-feedback-log.md).
2. Update [../calibration-pack/scorer-notes.md](../calibration-pack/scorer-notes.md) if scoring drift appeared.
3. Repair module files where learners got stuck for reasons the module should have solved.
4. Update [../release-pack/module-release-checklist.md](../release-pack/module-release-checklist.md).
