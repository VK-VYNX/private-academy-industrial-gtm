# Module 1 QA

This folder stores validation evidence for Module 1.

## Required Evidence

| Evidence | Owner | Status |
|---|---|---|
| automated validation report | academy operator | generated by `scripts/validate-academy.ps1` |
| Mermaid render report | academy operator | generated by `scripts/render-mermaid-flowcharts.ps1` |
| visual review log | academy operator and facilitator | maintained in `flowcharts/visual-render-qa.md` |
| cohort feedback log | facilitator | maintained in `cohort-ops/cohort-feedback-log.md` |
| scorer calibration notes | reviewer | maintained in `calibration-pack/scorer-notes.md` |
| release checklist | academy owner | maintained in `release-pack/module-release-checklist.md` |

## Validation Command

From the repo root:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\validate-academy.ps1
```

The command writes:

```text
modules/module-01-industrial-demand-generation/delivery-pack/qa/last-validation-report.md
```

## Release Rule

No Module 1 release should be approved without a current validation report and a human-reviewed visual QA log.
