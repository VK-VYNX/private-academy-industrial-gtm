# Exercise 05 - Content Plan Worksheet

Use this worksheet with Slide 22 of the Industrial Demand Generation workshop.

This exercise turns your top five MOIN priorities from Exercise 04 into a practical first-five content asset plan. The goal is not to create a content calendar. The goal is to decide which five assets can help buyers move, help sales act, and create useful account signals in the next 90 days.

## Exercise Outcome

By the end of this exercise, you will have:

- Five priority content or buyer-enablement assets;
- Demand state, buyer role, and buyer question for each asset;
- Proof or SME input required for each asset;
- Sales use and initial distribution path for each asset;
- Signal created by engagement with each asset;
- Build, Repair, Use As-Is, Later, or Do Not Build decision for each asset;
- Owner for each asset.

The output becomes the First-Five Content Asset Plan section of your Industrial Demand Generation strategy.

## How to Complete This Exercise

1. Copy the top five MOIN priorities from Exercise 04.
2. Choose one asset type for each priority.
3. Choose the best asset format.
4. Identify the proof or SME input required.
5. Choose how sales will use the asset.
6. Choose an initial distribution path.
7. Choose the signal the asset should create.
8. Decide whether each asset should be Built Now, Repaired, Used As-Is, Delayed, or Not Built.

Use exactly five assets. This keeps the exercise operational and prevents a bloated content calendar.

## First-Five Asset Quick Guide

| Asset Type | Use When |
|---|---|
| Problem-Risk Asset | Buyers do not yet understand the risk, cost, urgency, or operational consequence. |
| Selection Criteria Guide | Buyers need to compare approaches or avoid price-only evaluation. |
| Proof or Application Asset | Technical, quality, production, maintenance, or implementation risk blocks movement. |
| Champion Internal Pack | The visible contact needs help selling the case internally. |
| Vendor Validation Asset | Supplier credibility, TCO, support, procurement, or delivery confidence must be proven. |

## Asset Format Menu

| Format | Best For |
|---|---|
| Diagnostic Checklist | Content Demand or early problem framing. |
| Selection Criteria Guide | Solution Demand and approach comparison. |
| Technical Application Note | Engineering, quality, or operations validation. |
| Validation or Compliance Checklist | Quality, audit, safety, or regulated buying contexts. |
| TCO or Cost-of-Inaction Model | Finance, leadership, or commercial justification. |
| Implementation Roadmap | Production, maintenance, service, and adoption risk. |
| Supplier Comparison Worksheet | Procurement and vendor evaluation. |
| Champion Internal Summary | Internal consensus and buyer enablement. |

## Proof and SME Input Menu

| Proof or SME Input | Use When |
|---|---|
| Sales Feedback | The question appears repeatedly in conversations. |
| Technical SME Input | The asset explains fit, specification, application, or integration. |
| Quality or Compliance SME Input | The asset touches validation, documentation, audit, safety, or inspection. |
| Service or Maintenance Input | The asset explains uptime, spares, support, serviceability, or lifecycle risk. |
| Finance or Commercial Input | The asset includes TCO, payback, cost of delay, or margin logic. |
| Existing Case Evidence | There is credible prior experience, but it must be anonymized and generalized. |
| Proof Gap | The asset should not be published until evidence is repaired. |

## Sales Use Menu

| Sales Use | Use When |
|---|---|
| Educate Early-Stage Account | Buyer is still learning the problem. |
| Open Role-Specific Conversation | Sales needs a useful reason to engage a role. |
| Support Technical Validation | Engineering, production, quality, or maintenance needs proof. |
| Equip Champion Internally | Main contact must circulate the case. |
| Handle Repeated Objection | A predictable blocker keeps slowing deals. |
| Re-Engage Stalled Account | Interest exists but movement has stopped. |
| Support Procurement or Finance Review | Commercial confidence is required. |

## Initial Distribution Path Menu

Keep this light. Exercise 06 will build the full distribution plan.

| Path | Use When |
|---|---|
| Sales Share | Asset is useful in active or target-account conversations. |
| SME-Led Post or Explanation | Expert credibility is needed. |
| Event or Webinar Follow-Up | The asset answers questions from a field interaction. |
| Partner, Distributor, or Advisor Share | Ecosystem roles influence access or recommendation. |
| Website or Resource Hub | Buyers need an easy reference path. |
| Association or Industry Group | Trust is stronger through an industry context. |
| Paid or Targeted Amplification | Narrow audience targeting is required. |

## Signal Created Menu

| Signal | Meaning |
|---|---|
| Repeat Engagement by Target Account | Learning is becoming account-level interest. |
| Asset Shared Internally | Buyer enablement may be happening. |
| Role-Specific Question Asked | A real committee concern is visible. |
| Technical Follow-Up Requested | Solution Demand is increasing. |
| TCO or Procurement Question Asked | Commercial evaluation is starting. |
| Multi-Role Engagement | Committee coverage is improving. |
| Sales Conversation Created | Asset created useful account action. |
| Weak Signal Only | Continue nurture; do not over-route. |

## Build, Repair, or Later Decision Menu

| Decision | Use When |
|---|---|
| Build Now | The asset does not exist and is needed for the 90-day pilot. |
| Repair | A usable asset exists but needs role, proof, sales use, or signal clarity. |
| Use As-Is | The asset is already credible, specific, and sales-usable. |
| Later | The asset is useful but not essential for the next 90 days. |
| Do Not Build | The asset is generic, weak-fit, unsupported, or not tied to movement. |

## Quality Check

Use this check before accepting the content plan.

| Check | Pass Standard |
|---|---|
| MOIN Link | The asset comes from a top MOIN priority. |
| Buyer Role | The asset serves a real buying role. |
| Demand State | The asset is clearly Content, Solution, or Vendor Demand. |
| Proof | The asset has credible evidence or a named proof gap. |
| Sales Use | Sales can use it without rewriting it from scratch. |
| Distribution Readiness | The asset has at least one realistic initial path. |
| Signal Value | Engagement would produce useful account evidence. |

Reject assets that sound polished but do not change buyer belief, sales action, or account routing.
