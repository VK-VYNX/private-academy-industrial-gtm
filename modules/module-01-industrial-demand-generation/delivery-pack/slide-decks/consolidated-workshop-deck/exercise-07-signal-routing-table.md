# Exercise 07 - Signal Routing Table Worksheet

Use this worksheet with Slide 30 of the Industrial Demand Generation workshop.

This exercise turns signals from your distribution and demand-capture system into operational routing decisions. The goal is not to create a sales task for every engagement. The goal is to decide what each signal proves, what it does not prove, and what should happen next.

## Exercise Outcome

By the end of this exercise, you will have:

- Five routed industrial demand signals;
- Source class for each signal;
- Account fit, buyer role, demand state, and signal strength;
- Progression state for each account;
- Owner and SLA for each signal;
- Next action for each signal;
- False-positive risk for each signal;
- Missing evidence before escalation;
- Opportunity review criteria;
- RevOps fields or tags required for tracking.

The output becomes the Signal-Routing Table section of your Industrial Demand Generation strategy.

## How to Complete This Exercise

1. Select exactly five signals from the Signal Set Requirement.
2. Use the Signal Starter Menu if you do not have clean real signals ready.
3. Classify each signal by source class.
4. Assign account fit before choosing any action.
5. Identify the buyer role or mark it as Unknown.
6. Assign demand state and signal strength.
7. Choose the correct progression state.
8. Assign owner, SLA, next action, false-positive risk, and missing evidence.
9. Complete the opportunity review criteria only for signals that may deserve sales activation or opportunity review.

Do not route weak engagement as an opportunity. Do not assign sales activation unless fit, role, demand state, signal strength, and next action are credible.

## Signal Set Requirement

Route exactly five signals.

| Required Signal Type | Purpose |
|---|---|
| One Weak Content Signal | Prevent premature sales activation. |
| One Moderate Content or Solution Signal | Practice research or committee mapping. |
| One Strong Vendor Signal | Practice sales activation or opportunity review. |
| One Field, Event, Partner, or Distributor Signal | Capture ecosystem evidence properly. |
| One False-Positive or Low-Fit Signal | Practice downgrade, disqualification, or low-touch nurture. |

## Signal Starter Menu

| Signal | Typical Interpretation |
|---|---|
| One Educational Asset View | Weak Content Demand unless repeated or role-specific. |
| Repeat Engagement With Problem-Risk Asset | Moderate Content Demand if account fit is strong. |
| Selection Criteria Guide Visit | Moderate Solution Demand. |
| Technical Application Note Requested | Solution Demand, possibly strong if role and account fit are clear. |
| Validation or Compliance Checklist Requested | Solution or Vendor Demand depending on role and timing. |
| Supplier Documents Requested | Vendor Demand if account fit and need are credible. |
| Technical Consultation Requested | Strong Vendor Demand if role, fit, and need are credible. |
| RFQ, Sample, Audit, or Plant Visit Requested | Strong Vendor Demand and possible opportunity review. |
| Distributor Flags Specific Project Need | Strong ecosystem signal if details are specific. |
| Event Conversation With Named Project Detail | Moderate to strong signal depending on role and evidence. |
| Plant Expansion or Capacity Signal | Moderate third-party signal until validated. |
| Generic Post Like or Badge Scan | Weak signal and high false-positive risk. |
| Poor-Fit Account Downloads Asset | Low-touch nurture or disqualification. |

## Source Class Menu

| Source Class | Use When |
|---|---|
| First-Party | Website, resource hub, form, webinar, or owned asset engagement. |
| Sales-Discovered | Field visit, sales call, service note, referral, or direct account conversation. |
| Ecosystem | Distributor, partner, consultant, advisor, association, or event signal. |
| Third-Party | Plant expansion, tender, certification, hiring, export, or market trigger. |

## Account Fit Menu

| Fit | Use When |
|---|---|
| High Fit | Account clearly matches the selected ICP focus. |
| Medium Fit | Account may fit, but segment, need, or economics require research. |
| Low Fit | Account is outside the best 90-day focus or weakly matched. |
| Disqualified | Account violates a disqualification rule from Exercise 02. |

## Buyer Role Menu

| Role | Use When |
|---|---|
| Champion | Visible contact may advance the case internally. |
| Technical Evaluator | Engineering, application, specification, or validation judgment is visible. |
| Production or Operations | Uptime, throughput, disruption, adoption, or process risk is visible. |
| Quality or Compliance | Audit, validation, documentation, inspection, or compliance concern is visible. |
| Maintenance or Service | Serviceability, spares, uptime, lifecycle, or support risk is visible. |
| Procurement | Supplier comparison, documents, pricing, terms, or vendor onboarding is visible. |
| Finance | TCO, payback, budget, or commercial case is visible. |
| Leadership | Business priority, strategic project, risk, or capital decision is visible. |
| Partner, Distributor, Consultant, or Advisor | Ecosystem influence or recommendation path is visible. |
| Unknown | Role is not yet known and must be researched before escalation. |

## Demand State Menu

| Demand State | Use When |
|---|---|
| Content Demand | Buyer is learning the problem, risk, cost, or opportunity. |
| Solution Demand | Buyer is comparing approaches, requirements, implementation paths, or tradeoffs. |
| Vendor Demand | Buyer is evaluating supplier proof, TCO, support, procurement confidence, or delivery readiness. |

## Signal Strength Menu

| Strength | Use When |
|---|---|
| Weak | Single, broad, anonymous, old, generic, or low-context engagement. |
| Moderate | Repeated, role-relevant, ICP-fit, solution-focused, or research-worthy engagement. |
| Strong | Named role, clear need, timing, project, technical request, RFQ, consultation, or procurement action. |

## Progression State Menu

| Progression State | Use When |
|---|---|
| Cluster ICP | Account fits target profile but need is unknown. |
| Future Pipeline | Account shows awareness or engagement, but need is unconfirmed. |
| Active Focus | Account has need, trigger, stakeholder, timing, or relationship evidence. |
| Opportunity | Account has declared requirement, value logic, champion, and next step. |
| Disqualified or Low-Touch | Account fails fit, value, timing, access, or technical criteria. |

## Owner Menu

| Owner | Use When |
|---|---|
| Marketing or Content Owner | Signal needs nurture, education, or asset follow-up. |
| RevOps | Signal needs logging, enrichment, field creation, or cadence review. |
| Sales or Account Owner | Signal needs account research, committee mapping, or direct action. |
| SME or Technical Owner | Signal needs expert response, technical validation, or proof. |
| Partner or Channel Owner | Signal came through distributor, partner, consultant, advisor, or association. |
| Leadership Review | Signal may require opportunity review, resource allocation, or escalation. |

## SLA Menu

| SLA | Use When |
|---|---|
| Nurture Cycle | Weak Content Demand needs no immediate sales task. |
| Weekly Review | Research, committee mapping, or enrichment is needed. |
| 48 Hours | Strong Solution Demand or clear role-specific question needs action. |
| Same Day | Strong Vendor Demand, RFQ, consultation, plant visit, audit, or project detail appears. |
| Hold or Disqualify | Fit is weak, risk is high, or disqualification rule applies. |

## Next Action Menu

| Next Action | Use When |
|---|---|
| Educate or Nurture | Buyer is learning and signal is weak or early. |
| Research Account | Fit, role, trigger, or timing is unclear. |
| Map Committee | One role is visible but committee coverage is incomplete. |
| Enable Champion | Visible contact needs proof for internal discussion. |
| SME Follow-Up | Technical, validation, application, or proof issue needs expert response. |
| Sales Activation | Fit, role, state, and strength justify direct account action. |
| Opportunity Review | Requirement, value, champion, next step, and timing may be present. |
| Partner Qualification | Ecosystem signal needs validation before sales action. |
| Disqualify or Low-Touch | Fit is weak, timing is poor, or signal is not relevant. |

## False-Positive Risk Menu

| Risk | Use When |
|---|---|
| Generic Engagement | Signal is broad, anonymous, or not role-specific. |
| Poor ICP Fit | Account does not match the selected 90-day focus. |
| Unknown Role | Signal cannot be tied to a buying committee role. |
| Weak Source Trust | Source is vague, unverified, or secondhand. |
| Event Noise | Signal may be a badge scan, casual booth visit, or low-context meeting. |
| Distributor Ambiguity | Partner signal lacks account, role, project, timing, or need detail. |
| Consultant Learning | Advisor may be researching generally, not influencing an active project. |
| Procurement Price Shopping | Commercial request may not reflect technical or strategic fit. |
| Old or Isolated Signal | Signal is not recent or repeated. |
| No Next Action | Team cannot define a relevant next step. |

## Missing Evidence Menu

| Missing Evidence | Use When |
|---|---|
| ICP Fit | Segment, application, size, or economics are unclear. |
| Buyer Role | Contact role or committee influence is unclear. |
| Need or Trigger | Problem, project, urgency, or operating pain is unclear. |
| Timing | No buying, project, audit, capacity, or implementation timing is visible. |
| Proof Requirement | Required technical, quality, commercial, or service proof is unknown. |
| Committee Coverage | Other critical roles are invisible. |
| Value Logic | TCO, payback, risk, or commercial case is unclear. |
| Next Step | No useful action can be stated. |
| Source Reliability | Signal source needs validation. |

## Opportunity Review Criteria

Complete this section only for signals that may deserve sales activation or opportunity review.

| Criterion | Required Evidence | Present? |
|---|---|---|
| ICP Fit | Account matches selected ICP and avoids disqualification gates. |  |
| Named Stakeholder | At least one relevant buying role is visible. |  |
| Need or Trigger | Problem, project, audit, timing, expansion, or requirement is visible. |  |
| Demand State | Signal is strong Solution Demand or Vendor Demand. |  |
| Value or Project Potential | Commercial value, risk, or project importance is plausible. |  |
| Next Step | Sales can define a relevant next conversation or action. |  |
| Committee Risk | Missing roles, blockers, or proof gaps are known. |  |
| False-Positive Risk | Major false-positive risks have been named or reduced. |  |

## Quality Check

| Check | Pass Standard |
|---|---|
| Fit Before Action | Low-fit signals are not routed as sales opportunities. |
| Demand State Clarity | Each signal is Content, Solution, or Vendor Demand. |
| Role Visibility | Unknown roles trigger research, not blind activation. |
| Strength Discipline | Weak signals are not escalated as opportunities. |
| Owner | Every signal has a responsible function or person. |
| SLA | Strong signals have clear timing; weak signals do not create false urgency. |
| Next Action | Action is specific, not generic follow-up. |
| False-Positive Risk | Each signal names what could make it misleading. |
| Missing Evidence | Unknowns are visible before escalation. |
| RevOps Visibility | The signal can be logged and reviewed in cadence. |

## RevOps Translation

| Field | What to Track |
|---|---|
| Account | Account connected to the signal. |
| Segment or ICP Fit | Selected focus and fit level. |
| Buyer Role | Buying committee role or Unknown. |
| Source Class | First-Party, Sales-Discovered, Ecosystem, or Third-Party. |
| Asset or Context | What created or revealed the signal. |
| Demand State | Content, Solution, or Vendor Demand. |
| Signal Strength | Weak, Moderate, or Strong. |
| Progression State | Cluster ICP, Future Pipeline, Active Focus, Opportunity, or Disqualified. |
| Owner | Person or function responsible. |
| SLA | When action or review should happen. |
| Next Action | Educate, Nurture, Research, Map Committee, Enable, Activate, Review, Qualify, or Disqualify. |
| False-Positive Risk | Why the signal might mislead. |
| Missing Evidence | What must be checked before escalation. |
| Review Date | When the signal is inspected in weekly cadence. |

## Final Output

At the end of this exercise, you should have:

1. Five routed industrial demand signals;
2. Source class, account fit, buyer role, demand state, and strength for each signal;
3. Progression state, owner, SLA, and next action for each signal;
4. False-positive risk and missing evidence for each signal;
5. Opportunity review criteria for signals that may justify activation;
6. RevOps fields or tags required for tracking.
