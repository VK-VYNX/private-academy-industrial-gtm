# Scorer Notes

Use these notes to calibrate reviewer judgment before grading learner submissions.

## Expected Scores

| Example | Expected Score | Decision |
|---|---:|---|
| weak submission | 25-40 | reject |
| repairable submission | 60-78 | repair |
| field-ready submission | 85-95 | pass |

## Why The Weak Example Rejects

The weak example uses activity language instead of operating logic. It has no credible ICP, no committee depth, no demand-state routing, no RevOps model, and no signal discipline. It would cause sales to chase weak engagement and would teach the organization the wrong lesson.

## Why The Repairable Example Repairs

The repairable example has the right industrial category, buyer roles, and early structure. It is not safe for field use because signal routing, RevOps ownership, content-to-sales use, and metrics are still incomplete.

## Why The Field-Ready Example Passes

The field-ready example makes tradeoffs. It narrows the ICP, defines disqualification, maps committee concerns, separates demand states, connects content to distribution and sales use, routes signals by evidence strength, and gives RevOps a weekly operating model.

## Reviewer Calibration Questions

- Did the learner make a narrow enough ICP decision?
- Did the learner separate learning intent from buying intent?
- Can sales use the content without guessing?
- Can RevOps track the signal and account progression state?
- Would a technical buyer see useful proof?
- Did the learner identify false positives?
- Did the learner correct AI output instead of accepting it?
- Is the pilot ready for supervision, or only for another workshop?
