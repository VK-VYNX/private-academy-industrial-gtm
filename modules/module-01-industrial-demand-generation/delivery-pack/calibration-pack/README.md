# Calibration Pack

This pack helps facilitators and reviewers score Module 1 work consistently.

Before grading live learner work, reviewers should read all three sample submissions and then align on the scoring notes.

## Files

| File | Purpose |
|---|---|
| [weak-submission-example.md](weak-submission-example.md) | shows a submission that sounds active but is not operational |
| [repairable-submission-example.md](repairable-submission-example.md) | shows a useful draft with gaps that can be repaired |
| [field-ready-submission-example.md](field-ready-submission-example.md) | shows the standard for supervised implementation |
| [scorer-notes.md](scorer-notes.md) | calibrates scoring and reviewer discussion |

## Calibration Meeting

1. Each reviewer scores the weak example alone.
2. Compare scores and name the failure modes.
3. Repeat for the repairable example.
4. Repeat for the field-ready example.
5. Align on what earns pass, repair, and reject decisions.
6. Only then score live learner submissions.

## Non-Negotiable Scoring Rule

A capstone cannot pass if it lacks signal routing, weekly cadence, or disqualification logic, even if the content plan looks polished.
